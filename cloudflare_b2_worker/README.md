# Cloudflare B2 Worker - Secure Download Gatekeeper

A Cloudflare Worker reverse proxy that intercepts and validates download requests for Backblaze B2 files. It ensures that only authorized users with a valid JWT can access your private B2 content.

## Features

- **JWT Validation**: Verifies short-lived HMAC-SHA256 tokens generated by KJSWeb.
- **S3 Proxying**: Uses AWS Signature V4 (`aws4fetch`) to securely fetch and stream content from a Private Backblaze B2 bucket.
- **Header Forwarding**: Supports HTTP Range headers for fast parallel downloads (IDM, JDownloader).
- **Edge Performance**: Zero-buffering streaming through Cloudflare's backbone for maximum speed and low memory usage.

## Setup

1. **Install dependencies**:
   ```bash
   cd cloudflare_b2_worker
   npm install
   ```

2. **Configure wrangler.toml**:
   Make sure `B2_ENDPOINT`, `B2_REGION`, and `B2_BUCKET` are set to your Backblaze settings.

3. **Deploy from terminal**:
   ```bash
   npx wrangler deploy
   ```

## Secrets Configuration

After deployment, you must set these secrets in your Cloudflare dashboard (or via CLI):

```bash
# Backblaze credentials
npx wrangler secret put B2_APPLICATION_KEY_ID
npx wrangler secret put B2_APPLICATION_KEY

# The random string shared with KJSWeb appsettings.json
npx wrangler secret put JWT_SECRET
```

## JWT Payload Details

The worker currently expects a JWT with the following claims:
- `sub`: User ID
- `file`: The relative B2 path to the file (e.g., `posts/ID/file.zip`)
- `exp`: Standard JWT expiration

Generated links should look like:
`https://your-worker.workers.dev/download?file=posts/ID/file.zip&token=YOUR_JWT_HERE`

## Deployment Checklist

1. [ ] Correct `B2_BUCKET` name in `wrangler.toml`
2. [ ] Correct `B2_REGION` (e.g., `us-east-005`)
3. [ ] Identical `JWT_SECRET` in both KJSWeb and Cloudflare Worker.
4. [ ] Worker is set to "Public" (it uses the JWT as its internal security).
